Context

  • RoomProvider

    The `RoomProvider` component serves as a context provider for sharing state and functionality related to rooms, blogs, menu, and booking details throughout the application. Below is a summary of its structure and functionality: By utilizing the `RoomProvider`, components nested within it can easily access and modify shared state and functionality related to room booking and blog management.

    • Component Definition

      The `RoomProvider` component is a functional component that accepts `children` as props. It uses object destructuring to access `children`.

    • State Management

      Several state variables are initialized using the `useState` hook to manage different aspects of the application state:

      • - `blogs`: Manages the state of blog data.
      • - `rooms`: Manages the state of room data.
      • - `menu`: Manages the state of menu data.
      • - `adults`: Tracks the number of adults selected for booking.v
      • - `kids`: Tracks the number of kids selected for booking.
      • - `total`: Calculates the total number of persons (adults + kids) for booking.
      • - `loading`: Manages the loading state when filtering rooms based on the total number of persons.

    • Effect Hook

      An effect hook (`useEffect`) is used to recalculate the total number of persons (`total`) whenever the values of `adults` or `kids` change. This ensures that the total count stays up to date.

    • Handle Click Function

      The `handleClick` function is invoked when the user clicks on the booking button. It sets the `loading` state to `true` to indicate that the filtering process is in progress. It filters the room data based on the total number of persons (adults + kids) selected for booking (`total`). Only rooms with a maximum capacity greater than or equal to the total count are retained. After a simulated delay of 3 seconds (to mimic asynchronous operations), the filtered room data is set using the `setRooms` function, and the loading state is set back to `false`.

    • Context Provider

      The `RoomContext.Provider` component wraps around the `children`, providing access to the shared state and functions via the context API. The values provided by the context include `rooms`, `menu`, `adults`, `setAdults`, `kids`, `setKids`, `loading`, and `blogs`.

  • ScreenSizeContext

    The `ScreenSizeProvider` component is responsible for detecting the screen size and providing this information to its children components through a context named `ScreenSizeContext`. Below is a summary of its structure and functionality:

    • Component Definition

      `ScreenSizeProvider` is a functional component that takes `children` as props.

    • State Management

      - It manages two state variables:

      • - `isMobile`: Indicates whether the screen size is considered as mobile (max-width: 500px).
      • - `isMidMobile`: Indicates whether the screen size is considered as mid-mobile (max-width: 1000px).

    • Effect Hook

      • - Two `useEffect` hooks are used to listen for changes in screen size based on media queries.
      • - The first `useEffect` hook listens for changes in screen size corresponding to mobile devices (max-width: 500px).
      • - The second `useEffect` hook listens for changes in screen size corresponding to mid-mobile devices (max-width: 1000px).
      • - Each hook sets the initial state of the corresponding `isMobile` or `isMidMobile` variable based on the current match of the media query.
      • - They also define a callback function to update the state variables whenever the media query matches change.
      • - The callback functions are added as event listeners to the media queries, and they update the state accordingly.
      • - Finally, each hook removes the event listener when the component is unmounted to avoid memory leaks.

    • Context Provider

      The `ScreenSizeProvider` component wraps around its children components, providing access to the `isMidMobile` and `isMobile` states via the `ScreenSizeContext` context. The context value includes `isMidMobile` and `isMobile` states.
    By utilizing the `ScreenSizeProvider`, components within its subtree can easily access and react to changes in screen size, adjusting their layout and behavior accordingly.
